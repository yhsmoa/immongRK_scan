# 재고스캔 페이지 + 재고관리 하위메뉴 설계

> 작성: 2026-07-14 · 대상: 재고관리 메뉴 분리(재고스캔/전체재고) + 스캔 입력 페이지 신설
> 기존 `/stock`(전체재고) 및 API 무변경, 신규 페이지·엔드포인트만 추가

---

## 0. 목표

1. 헤더 **재고관리** 메뉴를 드롭다운으로 → **재고스캔**(`/stockScan`, 신규) + **전체재고**(`/stock`, 기존)
2. 재고스캔 페이지: 위치·바코드 스캔으로 재고를 모아서 한 번에 `rk_stocks`에 저장
3. 저장 시 **같은 위치 + 같은 바코드가 이미 DB에 있으면 수량 합산**

---

## 1. 메뉴 / 라우트

- `header.html`: 기존 `<a href="/stock">재고관리</a>` 단일 링크 →
  ```
  재고관리 (m-item 드롭다운, data-path="/stock,/stockScan")
    ├ 재고스캔  → /stockScan
    └ 전체재고  → /stock
  ```
- `server.js`: `app.get('/stockScan')` → `stockScan.html` 추가 (기존 `/stock` 그대로)

---

## 2. 화면 구성 (`stockScan.html`, 2분할)

```
┌───────────────────────────┬─────────────────────────────────────────┐
│  왼쪽 (입력)               │  오른쪽 (스캔 결과)          [저장]      │
│                           │                                         │
│  [ 위치 스캔        ]      │  위치 | 바코드 | 상품명 | 개수 | 삭제   │
│   (큰 입력폼)             │  ───────────────────────────────────    │
│                           │  A-01 | R123.. | 상품명…  |  3  | [삭제] │
│  [ 바코드 스캔      ]      │  A-01 | R456.. | 상품명…  |  1  | [삭제] │
│   (큰 입력폼)             │  ...                                    │
└───────────────────────────┴─────────────────────────────────────────┘
```

### 왼쪽 입력 동작
- **위치 입력폼**
  - 스캔 방식 = 바코드 입력 + Enter
  - **다시 이 입력폼을 선택(focus)하면 위치 값 초기화** (새 위치 스캔 대비)
  - Enter 처리되면 → 현재 위치 확정 + **바코드 입력폼으로 자동 focus 이동**
- **바코드 입력폼**
  - 위치 확정 후 자동 활성화(focus)
  - Enter 시: 현재 위치 + 바코드로 스캔 1건 반영 → 입력값 비우고 **focus 유지**(연속 스캔)
  - 위치가 아직 없으면 토스트 "위치를 먼저 스캔하세요"
  - 바코드로 `rk_inventories` 조회해 **상품명 자동 표시** (없으면 `-`, 그래도 추가 허용)

### 오른쪽 테이블
- 헤더: **위치 · 바코드 · 상품명 · 개수 · 삭제**
- **정렬: 스캔 순서 최신이 제일 위** (새 스캔은 맨 위로)
- **같은 위치+바코드 재스캔 → 기존 행 개수 +1 하고 맨 위로 이동** (행 폭증 방지, "개수" 컬럼 활용) ✅결정
- 태블릿 등 **좁은 화면에서 상품명 2줄 줄바꿈** (상품명 열만 `white-space:normal`, 나머지 `nowrap`)
- **개수 셀 클릭 → 숫자패드 모달**(터치용)로 수정 ✅결정 (아래 2-1)
- **삭제 열**: 행마다 [삭제] 버튼 → `confirm()` "삭제하시겠습니까?" → 확인 시 해당 행 제거
- 우측 상단 **[저장]** 버튼 → 아래 API 호출

### 2-1. 개수 숫자패드 모달 (태블릿 터치 대응)
- 개수 셀을 **탭/클릭하면 숫자패드 모달** 오픈 (contenteditable 아님 — 터치 오조작 방지)
- 구성: 큰 현재값 표시 + `1~9 · 0 · ←(지우기) · C(전체삭제)` 버튼 그리드 + `[취소] [확인]`
- 확인 시 해당 행 `qty` 갱신 후 테이블 갱신, 값이 1 미만이면 확인 비활성(또는 무시)
- 모바일 키보드가 아니라 화면 내 버튼 탭으로만 입력 (숫자 키만)

### 이탈 경고(전체재고와 일관)
- 저장 안 된 스캔 행이 있으면 페이지 이탈/닫기 시 `beforeunload` 경고

## 3. API (routes/rkStocks.js 에 추가, 기존 무변경)

| 엔드포인트 | 동작 |
|---|---|
| `POST /api/stocks/lookup` | body `{barcodes:[]}` → `[{barcode, name, skuId}]` (rk_inventories 우선, 없으면 rk_stocks.item_name). 스캔 시 상품명 조회용 |
| `POST /api/stocks/scan-save` | body `{items:[{location, barcode, qty}]}` → 위치+바코드로 rk_stocks 조회 → **있으면 qty 합산, 없으면 신규 insert**(sku_id/item_name 은 rk_inventories 에서 보강). `{ok, updated, inserted}` 반환 |

- 합산 로직은 기존 `/api/inbound/stock-complete` 의 rk_stocks 처리와 동일 패턴 재사용
- 재고 이력(rk_stock_histories)은 이번엔 기록 안 함 (기존 위치/수량 수정 API 와 동일하게 단순 반영)

---

## 4. 데이터 흐름 (스캔 1건)

```
위치 스캔(Enter) → currentLocation = "A-01" → 바코드 입력 focus
바코드 스캔(Enter) → lookup(barcode) → name
   → pending 에서 (A-01|barcode) 찾기
       있으면 qty+1 & 맨 위로 / 없으면 {위치,바코드,상품명,qty:1} 맨 위 추가
   → 바코드 입력 비우고 focus 유지
[저장] → POST scan-save(items=pending) → 성공 시 pending 비움 + 토스트
```

## 5. 건드리지 않는 것
- 기존 `/stock`(전체재고) 페이지·기능(위치 인라인수정, 수량 저장버튼), 기존 stocks API 전부
- rk_stocks 스키마 (신규 컬럼 없음 — location/barcode/qty/sku_id/item_name 그대로 사용)
